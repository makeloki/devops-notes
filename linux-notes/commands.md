cat vs less: Битва за оперативку

cat (concatenate)
Как работает: Вываливает весь текст файла в терминал сразу.
На логе в 10 Гб: Попытается прочитать весь файл. Это может «повесить» терминальную сессию, забить буфер вывода и, в худшем случае, вызвать проблемы с потреблением ресурсов, если терминал начнет пытаться отрисовать миллиарды строк.
Когда юзать: Только для маленьких конфигов или чтобы быстро «склеить» пару файлов.
less
Как работает: Открывает файл постранично. Она не загружает весь файл в оперативную память сразу, а читает только ту часть, которую вы видите на экране.
На логе в 10 Гб: Откроет его мгновенно. Вы сможете спокойно скроллить, искать по тексту (через /) и не бояться, что сервер «ляжет».
Когда юзать: Всегда, когда файл больше экрана.
Главное правило: Если файл весит больше пары мегабайт — забудь про cat, используй less.



CHMOD 755 and CHMOD 644 

Цифры в команде chmod — это права доступа, записанные в восьмеричной системе. Каждая из трех цифр отвечает за свою группу пользователей:
Первая цифра: Владелец файла (User).
Вторая цифра: Группа владельца (Group).
Третья цифра: Все остальные (Others).
Права доступа в цифрах (Octal Mode)
Каждая цифра — это сумма баллов для конкретной роли (Владелец | Группа | Остальные).
Права	Буква	Значение	Описание
Read	r	4	Чтение содержимого файла или просмотр списка файлов в папке.
Write	w	2	Изменение файла или создание/удаление файлов внутри папки.
Execute	x	1	Запуск файла как программы или вход в папку.
No Access	-	0	Никаких прав.

Как это складывается (Примеры)

Сумма	Права (rwx)	Что это значит?
7	4 + 2 + 1	Полный доступ (rwx)
6	4 + 2 + 0	Чтение и запись (rw-)
5	4 + 0 + 1	Чтение и выполнение (r-x)
4	4 + 0 + 0	Только чтение (r--)

Твои примеры для шпаргалки:
chmod 755: Я могу всё (7), остальные — только читать и заходить (5). Идеально для скриптов и папок.
chmod 644: Я могу читать и править (6), остальные — только читать (4). Стандарт для обычных файлов (тексты, картинки).

Все комбинации прав доступа (от 0 до 7)
Цифра	Буквы	Права доступа (что можно делать?)	Типичное применение
0	---	Нет прав. Полный запрет.	Защита от всех (кроме root).
1	--x	Только выполнение. (Нельзя даже прочитать файл).	Специфические скрипты.
2	-w-	Только запись. (Редкость, нельзя читать).	«Слепые» папки для загрузки (Drop Box).
3	-wx	Запись + выполнение.	Встречается крайне редко.
4	r--	Только чтение.	Конфиги, которые нельзя менять.
5	r-x	Чтение + выполнение.	Доступ в папки для всех.
6	rw-	Чтение + запись.	Обычные рабочие документы.
7	rwx	Полный доступ.	Свои папки и исполняемые скрипты.
Как это выглядит в команде (три цифры сразу):
chmod 777 — «Проходной двор». Все могут всё (плохая практика для безопасности).
chmod 700 — «Моя крепость». Только ты (владелец) имеешь доступ, остальные — ноль.
chmod 444 — «Только для чтения». Никто, включая тебя, не может случайно изменить файл.
