1. Что такое Docker?
Определение: Платформа для разработки, доставки и запуска приложений в изолированных средах — контейнерах.

Проблема: "У меня работает, а на сервере нет" (разница в версиях библиотек, ОС, конфигах).

Решение: Приложение упаковывается со всем окружением.

Отличие от VM:

VM: Тяжелая, эмулирует железо, запускает целую гостевую ОС.

Docker: Легкий, использует ядро хостовой ОС, изолирует только процессы.

2. Основные компоненты (Святая Троица)
Dockerfile — скрипт с инструкциями (рецепт).

Image (Образ) — слепок файловой системы и параметров запуска (готовое блюдо, которое нельзя изменить).

Container (Контейнер) — запущенный образ (процесс в системе).

3. Анатомия Dockerfile
Основные команды, которые нужно знать:

FROM — базовый образ (например, python:3.10-slim).

WORKDIR — рабочая директория внутри контейнера.

COPY / ADD — копирование файлов с твоего компа в контейнер.

RUN — выполнение команд при сборке (установка пакетов).

CMD — команда, которая выполнится при запуске контейнера (только одна).

EXPOSE — указание порта, который слушает приложение.

4. Хранение данных (Volumes)
По умолчанию контейнеры эфемерны (удалил контейнер — удалил данные).

Volumes — механизм для хранения данных вне контейнера (базы данных, логи).

Bind Mounts — прямая привязка папки с твоего компа к папке в контейнере (удобно для разработки).

5. Docker Compose
Инструмент для запуска мульти-контейнерных приложений.

Описывается в docker-compose.yml.

Позволяет одной командой docker-compose up запустить сразу связку: App + DB + Redis.

Автоматически создает сеть между этими контейнерами.
