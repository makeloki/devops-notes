Практическая работа: Контейнеризация Python-приложения и оркестрация через Docker Compose
1. Подготовка окружения и структуры
Первым делом мы создали изолированную среду для нашего приложения внутри репозитория devops-notes.

Структура папок: Создана директория docker-study/python-app. Это важно для разделения контекста сборки: Docker при сборке видит только те файлы, которые лежат в указанной папке.

Файлы приложения:

app.py: Код на Flask, который выводит приветствие и считает хиты.

requirements.txt: Список зависимостей (flask, redis) для пакетного менеджера pip.

2. Написание Dockerfile
Мы создали Dockerfile — инструкцию для сборки образа.

Основные этапы сборки:

FROM python:3.10-slim: Выбор легкого базового образа.

WORKDIR /app: Установка рабочей директории внутри контейнера.

COPY requirements.txt .: Копирование списка зависимостей.

RUN pip install...: Установка библиотек (слой кешируется, если файл не менялся).

COPY . .: Копирование остального кода.

CMD: Определение команды, которая запустит сервер при старте контейнера.

3. Сборка и запуск одиночного контейнера
На этом этапе мы столкнулись с первыми трудностями (пути, расширения файлов и права доступа).

Сборка образа:

Bash
docker build -t python-app .
Важно: Точка в конце указывает на текущую директорию как контекст сборки.

Запуск:

Bash
docker run -d -p 8080:5000 --name my-python-container python-app
Здесь мы пробросили порт 8080 нашей машины на порт 5000 внутри контейнера.

4. Оркестрация через Docker Compose
Чтобы добавить базу данных Redis, мы перешли от ручного управления к Docker Compose.

docker-compose.yml: Создан файл манифеста, который описывает сразу два сервиса: web (наш питон) и redis (готовый образ из Docker Hub).

Сетевое взаимодействие: Docker Compose автоматически создал сеть, в которой Flask-приложение смогло обращаться к базе по имени хоста redis.

Команда запуска всей системы:

Bash
docker compose up -d
Эта команда одной строчкой собирает образы, создает сети и запускает все связанные контейнеры в правильном порядке.

Итог
Мы успешно развернули микросервисную связку. Счетчик посещений в браузере подтвердил, что данные успешно передаются между изолированными контейнерами.
